# API Specifications

Define your API contracts here using OpenAPI 3.0 specification.

## Structure

- `openapi.yaml` - Main API specification
- `schemas/` - Reusable data schemas
- `paths/` - API endpoint definitions

## Benefits of Spec-First

1. **Contract-First**: Frontend and backend teams work in parallel
2. **Documentation**: Auto-generated API docs
3. **Validation**: Request/response validation
4. **Code Generation**: Generate client SDKs and server stubs
5. **Testing**: Mock servers for development

## Tools

- Swagger Editor: https://editor.swagger.io/
- Postman: Import OpenAPI specs
- AWS API Gateway: Direct import support
