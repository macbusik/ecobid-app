# Architecture Decision Records

Document key architectural decisions here using the ADR format:

## Template

```markdown
# ADR-XXX: [Title]

Date: YYYY-MM-DD
Status: [Proposed | Accepted | Deprecated | Superseded]

## Context
What is the issue we're facing?

## Decision
What decision did we make?

## Consequences
What are the trade-offs?
```

## Example ADRs to Create

- ADR-001: Use serverless architecture on AWS
- ADR-002: Choose React Native vs Flutter for mobile
- ADR-003: API-first design with OpenAPI
- ADR-004: DynamoDB as primary database
- ADR-005: Authentication strategy (Cognito)
